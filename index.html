[01]

워크플로우 소개

이 문서는 Claude Code의 슬래시 커맨드, 백그라운드 훅 및 3-탭 대시보드를 연계하여 비즈니스 기획을 즉시 분석 가능한 구조의 프로덕션 코드로 전환해내는 자동화 파이프라인 지침서입니다. 복잡한 마케팅 요소를 배제하고 오직 규격, 파일 구조, 인터페이스 표준에만 집중합니다.

💡

모든 킷은 .claude/ 내부 파일들의 약속된 전제와 정적 해석 룰을 따라 작동합니다. 최종 산출물은 프로젝트 루트의 .output/ 폴더로 격리되어 정렬되며, .output/dashboard.html 을 브라우저로 열어 한눈에 추적할 수 있습니다.

[02]

[03]

기획 및 개발 명령어 레퍼런스

Command 실행 방식 주요 입력값 핵심 산출 파일 사용 용도
/pm [요청] Foreground (대화형) 자연어 기획안 idea/design.html, brief.md 유형 자동 감지(신규 vs 개선) 후 시각 산출물과 개발 입력 명세를 함께 생성. 승인 시 /dev 자동 실행
/dev [ID] Background 태스크 ID complete.md 작성된 brief.md를 로드해 백그라운드에서 구현·빌드 검증. 일반적으로 PM 승인 마커로 자동 실행
/do [작업] Background 버그 패치 등 자연어 지시 complete.md 기획 단계 없이 직접 코드 수정 및 에러 검증을 다이렉트 수행 (단발성 작업용)
[04]

Setup 명령어

프로젝트 초기 부트스트랩, Next.js 보일러플레이트 생성, 룰셋 검증, 사업계획서 초안 작성을 담당하는 전용 슬래시 커맨드 사양입니다.

실행 패턴: /setup:init 최초 1회 부트스트랩

프로젝트 루트를 스캔해 모든 에이전트(/pm, /dev, /do)가 참조할 단일 진실 소스인 CLAUDE.md 와 대시보드용 .output/ 디렉토리(meta.json · history.json · dashboard.html)를 초기화합니다.

- 빌드 파일 감지 (package.json / pom.xml / pyproject.toml / go.mod 등)
- 단일/멀티 프로젝트 자동 인식 → meta.json 의 projects[] 배열로 표현
- 코드 컨벤션 · 스타일 토큰 · 빌드 명령어 추출 후 CLAUDE.md 생성
- .claude/templates/dashboard-template.html → .output/dashboard.html 로 복사
실행 패턴: /setup:create-nextjs [프로젝트명] 프로덕션 전용 사양

빌드 보일러플레이트를 엔터프라이즈 사양(Tailwind, shadcn/ui, Zustand 상태 통합, TanStack Query, Pretendard 폰트 페어링, TS Strict 모드)에 맞추어 즉시 배포합니다.

- View-Action-Container (VAC) 레이어 자동 수립
- 404, error, loading 화면 구성 포함
- ESLint / Prettier 통합 빌드 최종 완결성 점검
실행 패턴: /setup:check-nextjs [--report | --fix] 검사 및 치유

활성화된 Next.js 프로젝트가 /setup:create-nextjs 의 아키텍처 및 테마 룰셋(Global CSS 변수 무결성, 공급자 인젝션 등) 검사 항목을 통과하는지 자가 채점합니다.

--report : 자동 조치 없이 위반 구조 리포트만 기록
--fix : 누락된 레이아웃 복원 및 빌드 차단 에러 자가 해결 시도
실행 패턴: /setup:onepager [주제] 기획 문서 생성

.claude/templates/onepager-template.html 을 기반으로 사업계획서 또는 프로젝트 원페이저를 자동 생성합니다. 시장·문제·솔루션·로드맵 구조를 표준화된 HTML 로 출력합니다.

[05]

대시보드 (3-탭 구조)

.output/dashboard.html 은 새로고침으로만 동기화되는 정적 HTML 입니다. 폴링·웹소켓 없음 — 페이지 로딩 시 history.json 과 meta.json 을 1회 fetch 한 뒤 모든 데이터를 확정합니다.

TAB 01

📊 개요

프로젝트 메타 배너 + 진행률 도넛 + 시간축 KPI (이번주 완료·평균 소요·정체 7일+) + 상태별 분포.

🔔 즉시 처리 필요

승인대기 / 7일+ 정체 태스크를 카드 형태로 노출. 승인대기 카드 클릭 시 /dev 명령 자동 복사.

TAB 02

📈 진행 내역

모든 태스크의 logs[] 를 평탄화한 통합 활동 스트림. 시간 역순으로 phase 별 색상 마커와 함께 표시.

필터

태스크 이름 검색 · phase(기획/디자인/개발/폐기) · 기간(오늘/7일/30일) 다중 필터링.

TAB 03

📁 산출물

태스크별 카드 그리드. 각 카드에 산출물 타입 칩(Idea/Design/Brief/Complete) + 상태 배지 표시.

메타 파싱

complete.md 의 변경 사항·미해결 이슈를 정규식 파싱해 ✓ N파일 / ⚠ N이슈 칩으로 노출.

🔄

동기화 모델: 새로고침 1회 = 단일 fetch

실시간 동기화(폴링/웹소켓/Observer)는 의도적으로 배제. 페이지 로드 시 Promise.allSettled 로 모든 complete.md 메타를 일괄 fetch 한 뒤 단일 렌더로 끝냅니다. 데이터가 갱신되었을 때는 사용자가 직접 새로고침.

commands/pm.md

기획 워크플로우 명세 (foreground)

사용자 질의를 설계 템플릿과 결합하여 idea/design.html + brief.md를 생성합니다. 승인 시 __AUTO_DEV__ 마커로 /dev 자동 트리거.

* 왼쪽 트리의 임의 파일을 선택하면 메타 데이터와 역할 서술이 연계 표시됩니다.
